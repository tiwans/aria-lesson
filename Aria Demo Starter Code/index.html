About ARIA

ARIA (Assistive Rich Internet Applications), is a spec from the W3C created to improve accessibility of applications by providing more context and more interactivity to HTML attributes. It was developed through the HTML Accessibility Taskforce, and is published by the HTML Working Group with approval by the Protocols and Formats Working Group.

ARIA's Markups:

  • Roles: Defines the element
  • Properties: Adds properties to the elements
  • States: Defines the current states of the element

What Does ARIA Solve?

  • In general, accessibility issues with rich internet applications can be characterized as:Inability to provide the semantic structure of page areas and functionality (e.g., navigation, main content, search, etc.)
  • Inaccessibility of content that is dynamic and may change within the page (e.g., AJAX content updates)
  • Inability to change keyboard focus to page elements (e.g., setting focus to an error message within the page)
  • Difficulty with keyboard and screen reader accessibility with complex widgets and navigation elements (e.g., sliders, menu trees, etc.)

What Does ARIA NOT Solve?

WAI-ARIA is not intended to influence browser behavior. Unlike a real button element. The browser itself does not know that a div with role of “button” is a button, only its accessibility API portion does.

NOTE: As a consequence, this means that you absolutely have to implement keyboard navigation, focusability and other behavioural patterns known from desktop applications yourself.
